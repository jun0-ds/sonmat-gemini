# benchmark 스킬

## 개요

루프 실행 결과의 정량 비교·평가 프레임워크. 코드, 글쓰기, 데이터 분석 등
모든 도메인의 루프에서 사용 가능하며, 두 가지 이상의 접근법·설정·규율 조합을
같은 태스크에 적용해 어떤 쪽이 더 나은지 객관적으로 판단한다.

---

## 측정 항목

| 항목 | 측정 방법 |
|------|-----------|
| 결과물 품질 | 같은 태스크 결과를 사용자가 직접 비교 |
| 토큰 사용량 | 세션 종료 시 API 사용량 기록 |
| 소요 시간 | 세션 시작~완료 타임스탬프 |
| 규율 준수율 | 워커 로그에서 규율 준수 여부 |
| 불필요한 반복 | Fluency break로 인한 헛도는 횟수 |

---

## 결과 기록 형식

실험 결과는 아래 경로에 마크다운으로 저장한다.

```
benchmark/results/YYYY-MM-DD-{task-name}.md
```

### 템플릿

```markdown
# 비교실험: {태스크명}

## 환경
- 태스크:
- 도메인:
- 복잡도:

## 결과

| 항목 | A | B |
|------|---|---|
| 품질 | | |
| 토큰 | | |
| 시간 | | |

## 판단
- keep / discard / refine
- 근거:
```

---

## 실험 절차

1. **대상 선정** — 비교할 두 가지 이상의 설정(규율 조합, 에스컬레이션 임계값, 루프 전략 등)을 명시한다.
2. **동일 조건 확보** — 같은 태스크·입력 데이터를 각 설정에 동일하게 제공한다.
3. **독립 실행** — 각 설정을 독립 세션에서 실행해 교차 오염을 방지한다.
4. **지표 수집** — 위 측정 항목 표에 따라 수치를 기록한다.
5. **결과 저장** — 템플릿을 채워 `benchmark/results/` 에 저장한다.
6. **판단** — keep / discard / refine 중 하나를 선택하고 근거를 한 줄로 명시한다.

---

## 자기 개선 루프

```
비교실험 결과
    ↓
약점 발견 (낮은 품질 / 높은 토큰 / 규율 이탈 등)
    ↓
규율·에스컬레이션·루프 전략 튜닝
    ↓
재비교 → 개선 확인
```

이 루프를 반복함으로써 sonmat 플러그인 전체의 실행 품질이 점진적으로 향상된다.
개선 사항은 해당 스킬의 SKILL.md 또는 글로벌 규율 파일에 반영한다.

---

## 주의사항

- 비교 조건이 둘 이상 다르면 결과 해석이 모호해진다. **변수는 하나씩** 바꿀 것.
- 품질 비교는 주관이 개입하므로, 가능하면 루브릭(채점 기준)을 미리 정의한다.
- 실험 결과가 "무승부"일 때는 토큰·시간 지표를 우선 기준으로 삼는다.
