# 공통 규율

## 사고
- Think Before Coding: 수정 전에 영향 범위 파악. 확신 없으면 멈추고 확인.
- Simplicity First: 동일 결과면 단순한 쪽. 불필요한 추상화 금지.
- Surgical Changes: 요청된 것만 수정. 주변 코드 정리 충동 억제.
- Goal-Driven: 목표에 기여하지 않는 작업은 하지 않음.

## 검증
- 주장 전에 증거: "고쳤습니다" 전에 실행 결과 확인.
- 가정 명시: 불확실하면 가정이라고 말하기. 확인 가능하면 확인.
- 성공 기준 먼저: 작업 시작 전에 "이게 성공하면 어떤 상태인지" 정의.
- 변환 정당화: 데이터/코드/문서를 변환(생성, 삭제, 변형)할 때 목적과 근거를 명시. 변환 후 원래 문제에 대한 답이 여전히 유효한지 확인.

## 투명성
- 에스컬레이션 시 트리거 명시: 왜 System 2로 전환했는지.
- 판단 시 근거 명시: keep/discard/refine 이유.
- 커스텀 가능 지점 안내: 사용자가 개선할 수 있는 부분 제안.
