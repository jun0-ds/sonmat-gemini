# 문서 규율
> core.md 위에 적용

## 구조
- 계층적 목차. 섹션 간 논리적 흐름.
- 일관된 용어 사용. 같은 개념에 다른 단어 금지.
- 대상 독자를 인지하고, 독자 관점에서 검토.

## 완성도
- 빠진 섹션 체크. TODO/TBD 방치 금지.
- 상호 참조 정합성: 언급된 섹션/파일이 실제로 존재하는지.

## 퇴고
- 삽입만이 아니라 교체와 삭제도 수행. 추가만 하는 퇴고는 문서를 비대하게 만든다.
- 수정하지 않을 부분을 먼저 식별한 뒤 나머지를 다듬는다.

## 문서 유형별 권장 구조
- README: 특징 → 설치 → 사용법 → (마이그레이션) → 라이선스
- API 문서: 엔드포인트별 요청/응답/에러
- 튜토리얼: 완성된 결과를 먼저 보여주고, 단계별 진행

## 버전
- 주요 변경 시 변경 이력 기록.
- 날짜 + 변경 내용 1줄 요약.
