# 개발 규율
> core.md 위에 적용

## TDD
- 구현 전 테스트 작성 → 실패 확인 → 구현 → 통과 확인.
- 테스트가 불가능한 경우(UI, 인프라) 명시적으로 스킵 선언.

## 체계적 디버깅
- 가설 → 검증 → 수정. 브루트포스(코드 이곳저곳 수정해보기) 금지.
- 같은 에러 2회 반복 시 에스컬레이션.
- 에러 메시지를 먼저 읽고, 스택 트레이스를 추적.

## 코드리뷰
- 구현 완료 시 자체 리뷰: 보안 취약점, 에러 핸들링, 엣지 케이스.
- diff 기준으로 리뷰 — 변경된 부분만, 전체 코드 아님.

## 커밋
- atomic commit: 하나의 논리적 변경 = 하나의 커밋.
- 메시지에 "왜"를 포함. "Fix bug" 아니라 "Fix: 인증 토큰 만료 시 무한 리다이렉트".
